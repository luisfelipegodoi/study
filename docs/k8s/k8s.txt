Contexto e “estado geral”

kubectl config get-contexts
kubectl config use-context <ctx>
kubectl get nodes -o wide
kubectl get ns
kubectl get pods -A


Aplicar / remover manifests

kubectl apply -f manifest.yaml
kubectl apply -k ./kustomize-dir
kubectl delete -f manifest.yaml
kubectl diff -f manifest.yaml


Inspecionar recursos

kubectl get deploy,rs,po,svc,ing -n <ns>
kubectl describe pod <pod> -n <ns>
kubectl get events -n <ns> --sort-by=.metadata.creationTimestamp


Logs e troubleshooting

kubectl logs -f <pod> -n <ns>
kubectl logs -f <pod> -c <container> -n <ns>
kubectl exec -it <pod> -n <ns> -- sh
kubectl port-forward -n <ns> svc/<svc> 8080:80


Rollout (deploys)

kubectl rollout status deploy/<deploy> -n <ns>
kubectl rollout history deploy/<deploy> -n <ns>
kubectl rollout undo deploy/<deploy> -n <ns>


Services / endpoints / DNS

kubectl get svc -n <ns>
kubectl get endpoints -n <ns>
kubectl get endpointSlice -n <ns>
kubectl get ingress -n <ns>


Config e secrets

kubectl get cm,secret -n <ns>
kubectl describe cm/<cm> -n <ns>
kubectl create secret generic mysecret --from-literal=key=value -n <ns>


Pega recurso e salva

kubectl get <kind> <name> -n <ns> -o yaml > dump.yaml
kubectl get <kind> -n <ns> -o wide


Esperar Condicao

kubectl wait --for=condition=Ready pod -l app=myapp -n <ns> --timeout=120s
kubectl wait --for=condition=Available deploy/myapp -n <ns> --timeout=120s


Filtrar por label

kubectl get po -n <ns> -l app=myapp
kubectl describe po -n <ns> -l app=myapp
